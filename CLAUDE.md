# CLAUDE.md — NTLSN (www.ntlsn.com)

## What this project is

NTLSN — the National Teaching and Learning Sector Navigator — is a free, open hub mapping the Australian higher education teaching & learning sector: every symposium, workshop and PD opportunity across all 42 universities. One page, one portal. Built and maintained by Dr Seb Dianati (UniSQ). Sector-positioning matters: the site's credibility rests on the LIVE / FREE / OPEN-SOURCE claims on the hero, the Acknowledgement of Country / "Unceded Lands" elements, and accurate, verified event data.

## Current architecture (read this before touching anything)

The production artefact is a **single-file build**: `index.html` (~1MB) containing:

1. One inline JSON-LD block (WebSite schema) + one Event ItemList JSON-LD (id `ntlsn-events-ld`, generated by `scripts/build-feeds.mjs`).
2. A **minified React bundle (~912KB inline)** — this is compiled output. The original source is NOT in this repo. All app data is embedded inside the bundle as JS array literals:
   - 43 institutions: `{id, name, abbr, city, state, lat, lng, group (Go8/ATN/IRU/RUN/Unaligned), tlUrl, traditionalCountry}`
   - events: `{id, title, uni, date, endDate, type, desc, url, verified}`
3. **~12 small inline patch scripts appended after the bundle**, each doing one job:
   - video lightbox (YouTube/Vimeo play on-site)
   - share-form submit → Google Apps Script endpoint (POST, `text/plain` body, form name `symposium-share`)
   - "Shared across the sector" feed ← same Apps Script endpoint (GET)
   - "Latest" feed ← a second Apps Script endpoint
   - Featured Talks carousel (hard-coded YouTube IDs)
   - favicon badges on conference cards (Google s2 favicons)
   - "Unceded Lands" caption under the map
   - skip-link (WCAG 2.4.1), icon-button labels (4.1.2), main landmark + link names (2.4.4) — these are **runtime a11y patches** compensating for issues in the bundle
   - `ntlsn-aim` section injection, section re-ordering script (`ntlsn-order` — defines the canonical section order via DOM surgery)
   - PWA service-worker registration (id `ntlsn-sw`)

**Implication:** do not attempt to hand-edit the minified bundle. Changes go in via (a) the data files + generators, (b) new/edited patch scripts, or (c) the longer-term source rebuild (see TASKS.md, Epic 1).

## Data layer (single source of truth)

- `data/universities.json` — 43 institutions, extracted from the bundle. Canonical.
- `data/events.json` — canonical events. Count & date range drift as `scripts/archive-events.mjs` prunes past events; do not hardcode a number in copy — check `data/events.json` directly or the live count on /data/events.json.
- `scripts/build-feeds.mjs` — regenerates `events.ics`, `feed.xml`, `events-ld.json`, `sitemap.xml` from the data files. **Run after every data edit.** Until the source rebuild lands, the bundle's embedded copy of the data and `data/*.json` can drift — treat `data/*.json` as canonical and re-embed via the rebuild, not by patching the bundle.

Event `type` values in use: `conference`, `workshop`, `symposium`, `webinar`, etc. `verified: true` means the URL and dates were human-checked.

## Deploy

- Static host (Cloudflare Pages / Netlify style): `_headers` (CSP, HSTS, caching, content types for `.ics`/`feed.xml`), `_redirects` (real-404 hygiene: known junk/bot paths and every unknown path return `/404.html 404`; `/` serves `index.html` by default). **There is NO SPA fallback** — the site is multi-page static with no client-side router, so unknown paths 404 (do not "restore" a `/* → /index.html 200` rule; it re-introduces homepage-in-disguise soft-404s).
- `sw.js` — network-first for navigations, stale-while-revalidate for same-origin assets, cache name `ntlsn-vN` (bump `N` on breaking asset changes).
- `manifest.webmanifest` — PWA install metadata, theme `#211B14`.
- Public endpoints now served: `/events.ics` (subscribable calendar), `/feed.xml` (RSS), `/data/events.json` + `/data/universities.json` (CORS-open JSON API).

## External dependencies

- Two Google Apps Script endpoints (share-form intake; "Latest" feed). They are rate-limited and slow-cold-start — never block render on them; both patch scripts already fail soft. If replacing, the moderation workflow (Google Sheet review → approved rows surface in feed) must be preserved.
- Google s2 favicon service for conference logos.
- YouTube/Vimeo embeds (CSP `frame-src` already allows both).
- LTI 1.3 Deep Linking (`netlify/functions/lti-login.js`, `lti-launch.js`, `lti-jwks.js`) — lets an LMS embed links to the free commons. Deep Linking only (no Resource Link/NRPS/AGS — never reads institutional data by design). Platform registry: `data/lti-platforms.json`; deep-linkable content registry: `data/lti-content.json`. Setup: `docs/lti-setup.md`. Needs env `NTLSN_LTI_PRIVATE_KEY` (`scripts/generate-lti-keypair.mjs`). Status: Phase A+B (picker + signed LtiDeepLinkingResponse live, pending real-LMS end-to-end test).

## Conventions

- Australian English throughout (organise, programme where appropriate, no "Fall semester").
- Acknowledgement of Country and traditional-country data are not decorative — never strip, abbreviate or reorder them for layout convenience.
- Earthy "Eucalyptus & Clay" palette: bg espresso `#211B14`, sage `#8FB081`, terracotta `#C66C3F`, dusty plum `#A8737F`, gold `#C9A962`, amber `#E6A33C`, coral `#D96650` (de-neoned 2026-07-19 via PRs #71–#74; previously navy/teal/purple). Aboriginal flag accent (black/red/yellow) is used only in the Unceded Lands element.
- All new features must work with JS patch-script architecture OR the rebuilt source — do not create a third pattern.
- Accessibility floor: WCAG 2.2 AA. New code must not rely on the runtime a11y patch scripts.
- Every event added to `data/events.json` needs `verified: true` only after a human (Seb) confirms the URL resolves and dates match the organiser's page.

## Commands

```bash
node scripts/build-feeds.mjs            # regenerate feeds into repo root
node scripts/build-feeds.mjs --out dist # regenerate into a build dir
npm test                                # run the Netlify Functions test suite (node --test netlify/functions/test/*.test.js)
```

## What "done" looks like for any task

1. Feeds regenerated if data changed.
2. No regression to the LIVE/FREE/OPEN-SOURCE hero claims, Acknowledgement elements, or section order defined in `ntlsn-order`.
3. Lighthouse a11y ≥ 95; no new console errors; CSP not loosened without explicit sign-off.
4. No SPA routing assumptions — the site is multi-page static; unknown paths return a real 404 (there is no `/* → /index.html` fallback). New pages are real `.html` files.
